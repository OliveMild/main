## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.** Public disclosure before a fix is available can put users at risk.

### How to Report

To report a vulnerability, please [open a private security advisory](https://github.com/OliveMild/main/security/advisories/new) on GitHub. This ensures your report is handled confidentially.

### What to Include

To help us understand and resolve the issue quickly, please provide as much of the following information as possible:

- **Description**: A clear description of the vulnerability and its potential impact.
- **Reproduction steps**: Step-by-step instructions or a proof-of-concept script that demonstrates the issue.
- **Affected versions**: The version(s) of the project where the vulnerability exists.
- **Suggested fix**: Any mitigations or patches you have identified (optional but appreciated).

### What to Expect

- **Acknowledgement**: We will acknowledge receipt of your report within **48 hours**.
- **Updates**: We will keep you informed of our progress as we investigate and develop a fix.
- **Resolution**: We aim to release a fix or mitigation within **90 days** of confirming the vulnerability. For critical issues, we will expedite this timeline.
- **Credit**: With your permission, we will credit you in the release notes when the fix is published.

## Security Best Practices

When using this project:

- Always use the latest release to benefit from the most recent security patches.
- Review any scripts or code before executing them in sensitive environments.
- Report any unexpected or suspicious behavior even if you are unsure whether it constitutes a security vulnerability.
